## Module Descriptions

### main.py
**Purpose**: Application entry point and route handlers

**Contents**:
- FastAPI application instance
- CORS middleware configuration
- Route handlers for all endpoints
- Startup validation

**Key Functions**:
- `startup()`: Validates environment on startup
- `read_root()`: GET / endpoint
- `create_item()`: POST / endpoint
- `get_item()`: GET /items/{id} endpoint
- `get_items()`: GET /items/ endpoint
- `update_item()`: PUT /items/{id} endpoint
- `remove_item()`: DELETE /items/{id} endpoint
- `remove_all_items()`: DELETE /items/ endpoint

### config.py
**Purpose**: Application configuration and validation

**Contents**:
- Environment variable loading (.env file)
- Configuration validation upon startup

**Key Functions**:
- `validation()`: Validates required environment variables

**Environment Variables**:
- `DATABASE_URL`: Database connection string

### schemas.py
**Purpose**: Pydantic data validation schemas

**Contents**:
- TaskSchema: Validates task data for requests and responses

**TaskSchema Fields**:
- `id`: Optional integer (auto-generated by database)
- `title`: Optional string
- `description`: Optional string
- `status`: Required boolean

### sqlite.py
**Purpose**: Database models and CRUD operations

**Contents**:
- SQLAlchemy engine and session setup
- Task database model
- CRUD function implementations

**Database Connection**:
```python
db = create_engine(DATABASE_URL)
Session = sessionmaker(bind=db)
session = Session()
```

**Task Model**:
- Table name: `tasks`
- Columns: id (PK, auto-increment), title, description, status

**CRUD Operations**:
- `create(payload)`: Insert new task
- `read()`: Fetch all tasks
- `read_item(id)`: Fetch single task
- `update(id, payload)`: Update task
- `delete()`: Delete all tasks
- `delete_item(id)`: Delete single task

## Data Flow

### Create Task Flow
```
1. Client sends POST request with TaskSchema JSON
   └─> POST / {"title": "...", "description": "...", "status": true}

2. FastAPI validates payload against TaskSchema
   └─> Pydantic validation

3. create_item() handler processes request
   └─> Calls create(payload) from sqlite service

4. create() function creates Task object and persists to DB
   └─> Task(title=..., description=..., status=...)
   └─> session.add() and session.commit()

5. Response returned to client
   └─> {"details": "Item criado", "id": 1, "description": "...", "status": true}
```

### Read Tasks Flow
```
1. Client sends GET request
   └─> GET /items/ or GET /items/{id}

2. FastAPI routes to appropriate handler
   └─> get_items() or get_item(id)

3. Handler calls sqlite service
   └─> read() or read_item(id)

4. Service queries database with SQLAlchemy
   └─> session.query(Task).all() or session.query(Task).filter(...)

5. Results mapped to TaskSchema
   └─> Model field values converted to JSON

6. Response returned to client
   └─> JSON array or single object
```

### Update Task Flow
```
1. Client sends PUT request with updated TaskSchema
   └─> PUT /items/{id} {"title": "...", "description": "...", "status": ...}

2. FastAPI validates payload and extracts id

3. update_item(id, payload) handler processes request
   └─> Calls update(id, payload) from sqlite service

4. update() function:
   - Queries task by id
   - Updates all fields from payload
   - Commits changes

5. Response returned
   └─> {"details": "item atualizado", "id": 1, "description": "...", "status": ...}
```

### Delete Task Flow
```
1. Client sends DELETE request
   └─> DELETE /items/ or DELETE /items/{id}

2. FastAPI routes to handler
   └─> remove_all_items() or remove_item(id)

3. Handler calls sqlite service
   └─> delete() or delete_item(id)

4. Service deletes from database
   └─> session.query(Task).delete() or session.delete(object)
   └─> session.commit()

5. Response returned
   └─> {"message": "All items deleted"} or {"details": "item deletado", "id": 1}
```

## Technology Stack

### Core Framework
- **FastAPI** (v0.135.2+): Modern async Python web framework
  - Built on Starlette for HTTP handling
  - Automatic OpenAPI/Swagger documentation
  - Type hints and validation

### Database & ORM
- **SQLAlchemy** (v2.0.48+): Python ORM
  - Database abstraction layer
  - Support for multiple database backends
  
- **SQLite**: Default embedded database
  - Zero-configuration
  - File-based storage
  - Perfect for development

### Server
- **Uvicorn** (v0.42.0+): ASGI server
  - High-performance async server
  - Auto-reload capability
  - Production-ready

### Data Validation
- **Pydantic** (v2.13.1+): Data validation library
  - Runtime type checking
  - Automatic JSON serialization
  - Rich error messages

### Environment Management
- **python-dotenv** (v1.2.2+): Environment variable loading
  - Load .env files
  - Secure credential management

### Database Drivers
- **psycopg2** (v2.9.11+): PostgreSQL adapter
  - Included for potential future PostgreSQL support
  - Can be used without PostgreSQL backend

## Design Patterns Used

### 1. **Service Layer Pattern**
- Database operations isolated in `sqlite.py`
- Decouples routes from database logic
- Easy to swap implementations (e.g., switch from SQLite to PostgreSQL)

### 2. **Model-View-Controller (MVC)**
- **Model**: Task (database model)
- **View**: Response schemas (TaskSchema)
- **Controller**: Route handlers (main.py)

### 3. **Repository Pattern**
- CRUD operations encapsulated in service functions
- Consistent data access interface

### 4. **Dependency Inversion**
- FastAPI dependency injection for shared resources
- Startup validation ensures proper configuration

## CORS (Cross-Origin Resource Sharing)

The application includes CORS middleware configured for development:

```python
origins = [
    "http://localhost.com",
    "https://localhost.com",
    "http://localhost",
    "http://localhost:8080",
]
```

**Configuration**:
- `allow_origins`: Whitelisted domains
- `allow_credentials`: Allow cookies/auth headers
- `allow_methods`: All HTTP methods allowed
- `allow_headers`: All headers allowed

## Error Handling

### Validation Errors
- Pydantic automatically validates request bodies
- Returns 422 Unprocessable Entity with detailed error messages

### Startup Errors
- Environment variable validation in startup event
- Application fails to start if required vars missing

### Database Errors
- SQLAlchemy exceptions propagate
- Returns 500 Internal Server Error

### CORS Errors
- Browser blocks requests from non-whitelisted origins
- Middleware rejects requests from unauthorized sources
